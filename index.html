Lesson One

  1. Write one paragraph summarizing what semantic elements are (max of 150 words)
  2. Create a new git repository on your computer. Name it semantic-elements.
  3. Push your code to GitHub as a new repository.

Lesson Two

  1. Code a semantically structured HTML document using the following content sectioning.
  2. Create a new Git repository on your computer called “portfolio”. This will be the home base for all future assignments, and the pages you start coding here today will be progressively built upon for your final course project.

Lesson Three

  1. Create a directory called 'styles' and within it, stylesheet.css. In this stylesheet, write some styling for your portfolio site. This file must have at least 50 lines of code. Use your design skills or just experiment. This assignment is about using CSS properly, not design.
  2. Use the CSS validator to check the quality of your code.

Lesson Four

  1. Finish creating the code for the mockupLinks to an external site. we started in class. Getting your page to appear just like the provided mockup is NOT AS IMPORTANT as writing valid, semantic code.
  2. Create a second page for the site that mockup represents, build it with valid HTML and CSS, styling to match theme of mockup, using your best judgement.

Lesson Five

  1. Incorporate Google Fonts and Font Awesome in your portfolio site. Use these web fonts wherever and however you like to enhance your design. I will expect to see links to Google Fonts and Font Awesome stylesheets in the head of your pages (you can download and include the Font Awesome if you'd prefer), and at least one instance of using each in the HTML.
