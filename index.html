Lesson One

Click Here to View Lesson One

  1. Write one paragraph summarizing what semantic elements are (max of 150 words)
  2. Create a new git repository on your computer. Name it semantic-elements.
  3. Inside the repo, code an HTML document. Name it index.html.
  4. Use at least 15 different types of HTML elements (correctly!).
  5. View your document in a browser to ensure it displays as you expect.
  6. Push your code to GitHub as a new repository.

Lesson Two

Click Here to View Lesson two

  1. Read the articles listed on the Lesson 02 Overview page.
  2. Code a semantically structured HTML document using the following content sectioning.
  3. Create a new Git repository on your computer called “portfolio”. This will be the home base for all future assignments, and the pages you start coding here today will be progressively built upon for your final course project.
  4. In this portfolio directory, create a simple, two page site. The pages should link to each other in the main navigation. One of the pages should be a list of projects that you have done, each with a title, small image, and link to location of project. Be sure to include week 1's assignment! I know you might not have a lot of projects to link to right now- just use dummy data and links that go to “#” to flesh it out as necessary. The second page can be whatever you like, though "About Me" is a nice default. Whichever you would like to be the landing page for the site should be named index.html.
  5. Use dummy content wherever you like on these pages (feel free to use Lorem Ipsum. We'll be revisiting this project over the next several weeks, and you can add more content as we go. The focus should be on writing clean, semantic HTML. We will add CSS to this site starting next week. Please do not do any styling this week.
  6. There are no length requirements on these pages, but I expect each to have header, article, and footer sections, with relevant, semantic content within each. File names should be "kebab style" (lowercase, hyphens instead of spaces) and internal links should be correct. As always, style matters. If you are including local images, they should be inside a directory called "images" within the main project directory.
  7. Remember to view your pages in different browsers before turning this in. If you've made HTML errors that reduce the functionality of the page, you'll lose points, and they're easy to catch by just viewing it in a browser! You should also remember to always validate your code.
  8. Use git to commit as you go along. Create a GitHub repository called 'portfolio' and link it to your local repository. Push your work up to GitHub, and submit a link to the GitHub repo for the assignment. Refer to Lessons 00 and 01 if you need a refresher on git & GitHub.

Lesson Three

Lesson Three is Incomplete

  1. Create a directory called 'styles' and within it, stylesheet.css. In this stylesheet, write some styling for your portfolio site. This file must have at least 50 lines of code. Use your design skills or just experiment. This assignment is about using CSS properly, not design.
  2. Use the CSS validator to check the quality of your code.
  3. Include a link to the stylesheet in the head section of each of the html pages. This would also be a good opportunity to add more content to these HTML pages.
  4. Use git to commit your work as you go. Push your work up to your “portfolio” repository on GitHub master branch, and submit a link to the GitHub repo for the assignment. I will view the pages via your personal GitHub Pages URL and expect everything to be functional (no broken images or links, etc). Refer to Lesson 00 and 01 if you need a refresher on Git & GitHub.
